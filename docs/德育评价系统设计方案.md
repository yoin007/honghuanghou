# 高中学生德育评价管理系统 - 完整设计方案

## 一、系统概述

### 1.1 评价体系架构

```
┌─────────────────────────────────────────────────────────────┐
│                        德育评价系统                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌──────────────┐   ┌──────────────┐   ┌──────────────┐   │
│  │  数据输入    │ → │  数据分析    │ → │  评价赋分    │   │
│  └──────────────┘   └──────────────┘   └──────────────┘   │
│         ↓                  ↓                  ↓            │
│  ┌──────────────────────────────────────────────────────┐  │
│  │                    数据展示                          │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 四大模块定位

| 模块 | 功能 | 说明 |
|------|------|------|
| **数据输入** | 录入数据 | 日常表现、荣誉处分、任务完成 |
| **数据分析** | 统计分析 | 班级排名、事件统计、趋势分析 |
| **评价赋分** | 评分定级 | 计算总分、判定等级 |
| **数据展示** | 查询展示 | 个人报告、班级报表、导出打印 |

### 1.3 四大数据输入来源

| 输入类型 | 内容 | 分值影响 |
|---------|------|---------|
| 年级德育任务 | 按年级设定的必修德育项目 | 完成后加分 |
| 日常常规表现 | 积极事件 / 消极事件 | 加分 / 扣分 |
| 校级及以上事件 | 荣誉 / 处分 | 加分 / 扣分 |
| 集体事件 | 班级集体荣誉/活动 | 每人加分 |

### 1.4 计分规则

```
德育总分 = 基础分(60) + 日常表现分 + 校级事件分 + 任务完成奖励分
```

| 分值项 | 范围 |
|-------|------|
| 基础分 | 60分 |
| 积极事件 | +1 ~ +5 分 |
| 消极事件 | -1 ~ -5 分 |
| 校级荣誉 | +5 ~ +20 分 |
| 校级处分 | -5 ~ -30 分 |
| 任务完成 | +10 ~ +30 分/项 |

**注意：德育总分可超过100分**

### 1.5 等级评定

| 等级 | 分数范围 |
|------|---------|
| 优秀 | ≥ 90 分（无上限） |
| 良好 | 80 ~ 89 分 |
| 合格 | 60 ~ 79 分 |
| 不合格 | < 60 分 |

## 二、核心业务规则

### 2.1 学期数据流转

| 规则 | 说明 |
|------|------|
| 学期计算 | 每学期重新计算，历史数据保留 |
| 多学期统计 | 支持跨学期数据汇总查询 |
| 荣誉结转 | 不结转（记录保留，分值留在当学期） |
| 处分结转 | 不结转（记录保留，分值留在当学期） |
| 任务结转 | 未完成任务可结转，详见任务规则 |

### 2.2 任务规则（以学年为单位）

#### 任务周期

| 项目 | 说明 |
|------|------|
| 任务周期 | 以学年为单位（一个学年 = 两个学期） |
| 完成期限 | 一个学年内完成 |
| 结转时机 | 学年末处理 |

#### 任务结转规则

| 规则 | 说明 |
|------|------|
| 已完成任务 | 分值保留在完成学年，不结转 |
| 未完成任务 | 可结转到下一学年 |
| 结转分值 | 每次结转 ×60%（递减） |
| 最大结转次数 | 2次（高一→高二→高三） |
| 超过限制 | 任务作废，不得分 |

#### 任务结转示例

```
任务：技能考核A
原始分值：30分
原始学年：2025-2026学年（高一）

┌────────────────────────────────────────────────────────────────────┐
│ 学年          │ 状态   │ 结转次数 │ 当前可得分数 │ 操作           │
├────────────────────────────────────────────────────────────────────┤
│ 2025-2026     │ 未完成 │ 0        │ 30.00       │ 学年末结转     │
│ 2026-2027     │ 未完成 │ 1        │ 18.00       │ 学年末结转     │
│ 2027-2028     │ 完成   │ 2        │ 10.80       │ 得10.80分      │
└────────────────────────────────────────────────────────────────────┘

计算公式：30 × (0.6)² = 10.80分
```

### 2.3 集体事件处理

#### 集体事件类型

| 类型 | 说明 | 分值处理 |
|------|------|---------|
| 班级荣誉 | 班级获得集体荣誉（如先进班集体） | 每人加固定分 |
| 集体活动 | 班级参与集体活动（如运动会、合唱比赛） | 每人加固定分 |
| 集体违纪 | 班级集体违纪 | 每人扣固定分 |

#### 集体事件分配规则

```
班级集体荣誉示例：

事件：2025级1班获得"先进班集体"称号
分值：每人+5分
分配方式：全班所有在校学生

处理流程：
1. 创建集体事件记录
2. 自动为该班所有在校学生分配分数
3. 学生可标记是否实际参与（默认参与）
4. 未参与者可申请不加分
```

### 2.4 学生状态与班级调整

#### 年级与班级概念

```
级号（grade）: 入学年份，如 2025级、2024级
班级（class）: 如 2025级1班、2025级2班
年级等级: 高一/高二/高三（动态计算）

规则：
- 级号属于班级，班级永远属于某个级号
- 学生有级号归属（留级时会变）
- 自然升年级：级号不变，年级等级+1（高一→高二）
```

#### 级号与年级等级对应关系

| 级号 | 2025-2026学年 | 2026-2027学年 | 2027-2028学年 |
|------|--------------|--------------|--------------|
| 2025级 | 高一 | 高二 | 高三 |
| 2024级 | 高二 | 高三 | 毕业 |
| 2023级 | 高三 | 毕业 | - |

#### 学生状态

| 状态 | 说明 | 德育记录 | 参与统计 | 任务处理 |
|------|------|---------|---------|---------|
| 在校 | 正常就读 | 正常记录 | 参与 | 正常进行 |
| 休学 | 暂停学业 | 暂停记录 | 不参与 | 任务冻结 |
| 复学 | 恢复就读 | 恢复记录 | 参与 | 任务解冻继续 |
| 转入 | 外校转入 | 开始记录 | 参与 | 新分配任务 |
| 转出 | 离开本校 | 停止记录 | 不参与 | 任务作废 |
| 毕业 | 完成学业 | 结束记录 | 可查询历史 | 未完成任务作废 |

#### 休学期间任务冻结规则

| 规则 | 说明 |
|------|------|
| 休学时 | 未完成任务状态保持，不参与结转，计时暂停 |
| 复学时 | 任务解冻，继续计算，结转次数不变 |
| 冻结期间 | 任务不计入任何学年统计 |
| 复学后结转 | 按正常结转规则处理 |

#### 班级变更类型

| 变更类型 | 级号变化 | 班级变化 | 说明 |
|---------|---------|---------|------|
| 入学 | 设置 | 设置 | 新生入学 |
| 调班 | 不变 | 变 | 同级调班 |
| 留级 | 变大 | 变 | 去更晚入学的级（如2024级→2025级） |
| 复学 | 可能变 | 变 | 休学后复学，通常降级 |
| 转入 | 设置 | 设置 | 外校转入 |
| 转出 | 不变 | 离开 | 离开本校 |
| 休学 | 不变 | 不变 | 暂停学业 |
| 毕业 | 不变 | 不变 | 完成学业 |

#### 留级示例

```
学生A：2024级（高二）留级

当前状态：
- grade_id = 2024级（高二）

留级后：
- grade_id = 2025级（高一）
- class_id = 2025级某班

级号变化：2024 → 2025（变大）
原因：留级是去重新读低年级，低年级对应更晚入学的级号
```

### 2.5 处分与违纪处理

| 规则 | 说明 |
|------|------|
| 处分可撤销 | 撤销后扣分不返还 |
| 累进加重 | 同类违纪在指定时间内多次，额外扣分 |
| 警告不自动解除 | 到期后需手动处理 |
| 无申诉渠道 | 不提供申诉功能 |

#### 累进加重机制

```
示例：迟到

90天内违纪次数与扣分：
- 第1次：按原分值扣分
- 第2次：额外扣1分
- 第3次：额外扣3分
- 第4次及以上：额外扣5分
```

### 2.6 数据可见性

| 数据类型 | 可见角色 |
|---------|---------|
| 处分记录 | 管理员、学发部 |
| 消极事件 | 管理员、学发部、班主任 |
| 积极事件 | 管理员、学发部、班主任、教师 |
| 荣誉记录 | 全员可见 |
| 评价得分 | 管理员、学发部、班主任、学生、家长 |

**班主任额外限制：只能查看本班学生数据**

### 2.7 预警机制

| 规则 | 说明 |
|------|------|
| 扣分不通知家长 | 仅内部预警 |
| 预警对象 | 班主任、学发部 |

| 预警类型 | 触发条件 | 通知对象 |
|---------|---------|---------|
| 德育分过低 | 总分 < 50分 | 班主任、学发部 |
| 扣分过多 | 累计扣分 > 20分 | 班主任、学发部 |
| 违纪次数过多 | 学期内违纪 ≥ 5次 | 班主任、学发部 |

## 三、数据库设计


### 3.3 基础配置表

```sql
-- 学期表
CREATE TABLE semester (
    semester_id INT PRIMARY KEY AUTO_INCREMENT,
    semester_name VARCHAR(20) NOT NULL COMMENT '如：2025-2026上',
    year_id INT NOT NULL COMMENT '所属学年',
    start_date DATE NOT NULL,
    end_date DATE NOT NULL,
    status TINYINT DEFAULT 1 COMMENT '1=当前学期',
    FOREIGN KEY (year_id) REFERENCES school_year(year_id)
);

-- 学年表
CREATE TABLE school_year (
    year_id INT PRIMARY KEY AUTO_INCREMENT,

    UNIQUE KEY uk_enrollment_year (enrollment_year)

-- 年级等级配置表
CREATE TABLE grade_level_config (
    id INT PRIMARY KEY AUTO_INCREMENT,
    years_after_enrollment INT NOT NULL COMMENT '入学后第几年',
```

### 3.4 学生班级履历表

```sql
-- 学生班级履历（核心：记录学生在各时期的班级和级号归属）
CREATE TABLE student_class_history (
    id INT PRIMARY KEY AUTO_INCREMENT,
    student_id VARCHAR(20) NOT NULL,
    class_id INT NOT NULL COMMENT '班级ID',
    grade_id INT NOT NULL COMMENT '当时的级号',
    start_date DATE NOT NULL COMMENT '生效开始日期',

    FOREIGN KEY (student_id) REFERENCES student(student_id),
    FOREIGN KEY (class_id) REFERENCES class(class_id),
    FOREIGN KEY (grade_id) REFERENCES grade(grade_id),
    INDEX idx_student_date (student_id, start_date, end_date)
);

CREATE TABLE student_status_change (
    id INT PRIMARY KEY AUTO_INCREMENT,
    student_id VARCHAR(20) NOT NULL,

    from_class_id INT,
    to_class_id INT,
    from_grade_id INT,
    to_grade_id INT,
    effective_date DATE NOT NULL,
    reason TEXT,
    approver VARCHAR(20),
    created_at DATETIME DEFAULT NOW(),
    FOREIGN KEY (student_id) REFERENCES student(student_id)
);
```

### 3.5 事件配置表

```sql
-- 日常事件类型（含分数范围校验）
CREATE TABLE daily_event_type (
    event_id INT PRIMARY KEY AUTO_INCREMENT,
    event_name VARCHAR(50) NOT NULL,
    event_type TINYINT NOT NULL COMMENT '1=积极 2=消极',
    score INT NOT NULL COMMENT '加分/扣分',
```

```python
class ImportDependencyChecker:
    """导入依赖检查器"""

    DATA_STATUS = {
        'school_year': {'name': '学年配置', 'requires': []},
        'grade': {'name': '级号配置', 'requires': []},
        'semester': {'name': '学期配置', 'requires': ['school_year']},
        'class': {'name': '班级信息', 'requires': ['grade']},
        'student': {'name': '学生信息', 'requires': ['grade', 'class']},
    }

    @classmethod
    def get_all_status(cls, db) -> dict:
        """获取所有数据导入状态"""
        result = {}
        for key, info in cls.DATA_STATUS.items():
            table_map = {
                'school_year': 'school_year',
                'grade': 'grade',
                'semester': 'semester',
                'class': 'class',
                'student': 'student'
            }
            count = db.query(f"SELECT COUNT(*) as cnt FROM {table_map[key]}").first()['cnt']
            result[key] = {
                'name': info['name'],
                'completed': count > 0,
                'count': count
            }
        return result

    @classmethod
    def check_dependency(cls, db, data_type: str) -> dict:
        """检查某个数据类型的依赖状态"""
        if data_type not in cls.DATA_STATUS:
            return {'can_import': True, 'missing': []}

        dep_info = cls.DATA_STATUS[data_type]
        missing = []

        for req in dep_info['requires']:
            count = db.query(f"SELECT COUNT(*) as cnt FROM {req}").first()['cnt']
            if count == 0:
                missing.append({
                    'key': req,
                    'name': cls.DATA_STATUS[req]['name']
                })

        return {
            'can_import': len(missing) == 0,
            'missing': missing
        }


@router.get("/import/status")
async def get_import_status():
    """获取所有数据导入状态"""
    return ImportDependencyChecker.get_all_status(db)


@router.get("/import/check-dependency/{data_type}")
async def check_import_dependency(data_type: str):
    """检查数据导入依赖"""
    return ImportDependencyChecker.check_dependency(db, data_type)
```

## 十一、数据验证方案

### 11.1 验证检查项

| 检查项 | 检查内容 | 严重程度 | 说明 |
|-------|---------|---------|------|
| 基础数据完整性 | 学年、级号、班级、学生数量 | 错误 | 系统运行必要数据 |
| 外键关联完整性 | 学生→级号、履历→班级 | 错误 | 数据关联正确性 |
| 班级履历完整性 | 每个在校学生都有当前班级 | 错误 | 状态一致性 |
| 数据状态一致性 | 学生状态与履历匹配 | 警告 | 可能存在问题 |
| 事件类型完整性 | 必要的事件类型已配置 | 警告 | 影响录入功能 |

### 11.2 验证脚本

```python
def validate_data(db):
    """数据验证"""

    errors = []
    warnings = []

    print("=" * 60)
    print("德育评价系统数据验证")
    print("=" * 60)

    # 1. 基础数据数量检查
    print("\n[1] 基础数据检查")

    checks = [
        ("学年", "school_year", 3),
        ("级号", "grade", 3),
        ("学期", "semester", 6),
    ]

    for name, table, expected in checks:
        count = db.query(f"SELECT COUNT(*) as cnt FROM {table}").first()['cnt']
        status = "✅" if count >= expected else "⚠️"
        print(f"  {status} {name}: {count} 条")
        if count < expected:
            warnings.append(f"{name}数据不足")

    class_count = db.query("SELECT COUNT(*) as cnt FROM class").first()['cnt']
    student_count = db.query("SELECT COUNT(*) as cnt FROM student").first()['cnt']
    print(f"  {'✅' if class_count > 0 else '❌'} 班级: {class_count} 个")
    print(f"  {'✅' if student_count > 0 else '❌'} 学生: {student_count} 人")

    if class_count == 0:
        errors.append("班级数据为空")
    if student_count == 0:
        errors.append("学生数据为空")

    # 2. 外键关联检查
    print("\n[2] 外键关联检查")

    orphan = db.query("""
        SELECT COUNT(*) as cnt FROM student s
        LEFT JOIN grade g ON s.grade_id = g.grade_id
        WHERE g.grade_id IS NULL
    """).first()['cnt']
    print(f"  {'✅' if orphan == 0 else '❌'} 学生-级号关联: {orphan} 条异常")
    if orphan > 0:
        errors.append(f"{orphan}个学生级号关联异常")

    # 3. 班级履历检查
    print("\n[3] 班级履历检查")

    no_class = db.query("""
        SELECT COUNT(*) as cnt FROM student s
        LEFT JOIN student_class_history sch ON s.student_id = sch.student_id
            AND sch.end_date IS NULL
        WHERE s.status = '在校' AND sch.id IS NULL
    """).first()['cnt']
    print(f"  {'✅' if no_class == 0 else '❌'} 在校学生班级履历: {no_class} 人缺失")
    if no_class > 0:
        errors.append(f"{no_class}个在校学生缺少班级履历")

    # 4. 事件类型检查
    print("\n[4] 事件类型检查")

    positive = db.query("SELECT COUNT(*) as cnt FROM daily_event_type WHERE event_type=1 AND is_active=1").first()['cnt']
    negative = db.query("SELECT COUNT(*) as cnt FROM daily_event_type WHERE event_type=2 AND is_active=1").first()['cnt']
    print(f"  {'✅' if positive > 0 else '⚠️'} 日常积极事件: {positive} 种")
    print(f"  {'✅' if negative > 0 else '⚠️'} 日常消极事件: {negative} 种")

    if positive == 0:
        warnings.append("未配置日常积极事件类型")
    if negative == 0:
        warnings.append("未配置日常消极事件类型")

    # 5. 结果汇总
    print("\n" + "=" * 60)
    print("验证结果")
    print("=" * 60)

    if not errors and not warnings:
        print("✅ 所有检查通过，系统可以正常运行！")
    else:
        if errors:
            print(f"\n❌ 错误 ({len(errors)} 条):")
            for e in errors:
                print(f"   - {e}")
        if warnings:
            print(f"\n⚠️ 警告 ({len(warnings)} 条):")
            for w in warnings:
                print(f"   - {w}")

    return {'errors': errors, 'warnings': warnings}
```

## 十二、方案总结

| 模块 | 核心设计 |
|------|---------|
| **评分体系** | 基础分60 + 四类事件加减分 + 等级评定（可超100分） |
| **任务管理** | 以学年为单位，未完成任务可结转，每次×60%，最多2次 |
| **学期流转** | 荣誉/处分不结转，记录保留在原学期 |
| **班级管理** | 履历表记录历史，快照保证统计准确，留级级号变大 |
| **处分处理** | 可撤销不返分，累进加重机制 |
| **集体事件** | 班级集体荣誉/活动，每人统一加分 |
| **权限控制** | 基于角色的数据可见性配置 |
| **预警机制** | 内部预警，不通知家长 |
| **数据展示** | 个人报告、班级统计、年级对比、趋势分析、导出打印 |
| **数据完整性** | 唯一约束防重复、分数范围校验、状态枚举限制 |
| **批量操作** | 支持批量导入、模板下载 |
| **系统初始化** | 导入依赖检查、分步引导、自动创建班级履历 |
| **事件配置** | 界面化管理、分值约束、累进规则配置 |
| **数据验证** | 多维度检查、错误警告分级、验证报告 |
